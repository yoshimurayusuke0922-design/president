# Functional Specification

## 画面一覧

### 1. タイトル画面

表示項目:
- ゲームタイトル
- ルーム作成ボタン
- ルーム参加ボタン
- 必要ならルールボタン

### 2. ルーム作成 / 参加画面

表示項目:
- プレイヤー名入力
- ルーム作成ボタン
- ルームコード入力
- ルーム参加ボタン

### 3. 対戦設定画面（ホスト用）

設定可能項目:
- CPU人数
- CPU難易度（Easy / Normal / Hard）
- ローカルルール ON/OFF
  - 階段
  - 革命
  - 8切り
  - スペ3返し
  - 縛り
  - 5スキップ
  - 7渡し
  - 10捨て
  - 11バック
  - 12ボンバー
  - カード交換
- ゲーム開始ボタン

### 4. ルーム待機画面

表示項目:
- ルームコード
- 参加プレイヤー一覧
- ホスト表示
- CPU追加状況
- 適用ルール一覧
- ゲーム開始ボタン（ホストのみ）

### 5. 対戦画面

表示項目:
- 場のカード
- 現在のターンプレイヤー
- 革命中かどうか
- 11バック中かどうか
- 縛り中かどうか
- 残り手札枚数（全員分）
- 自分の手札
- 選択中カード
- 出すボタン
- パスボタン
- 接続状態表示
- ログ表示（任意）

### 6. 追加選択モーダル

以下のルール発動時に追加選択 UI を出す。

- 7渡し: 渡すカードを 1 枚選ぶ
- 10捨て: 捨てるカードを 1 枚選ぶ
- 12ボンバー: 全員に強制捨てさせる数字を選ぶ
- カード交換: 交換に出すカードを選ぶ

### 7. 結果画面

表示項目:
- 1位〜4位
- 各プレイヤー名
- CPU / 人間の区別
- もう一度遊ぶボタン
- タイトルへ戻るボタン

## プレイフロー

1. タイトル画面を開く
2. ホストがルーム作成、またはプレイヤーがルーム参加
3. ホストがルールと CPU 難易度を設定
4. プレイヤーが揃ったら不足人数を CPU で補う
5. ゲーム開始
6. 手札を配る
7. 開始プレイヤー決定
8. 順番に手を出す / パスする
9. 追加効果のあるルールが発動したら追加選択処理を行う
10. 順位が全て確定したら結果画面へ
11. カード交換ルール ON かつ連戦なら交換フェーズを挟んで次ゲームへ

## 入力仕様

- 自分の手札から複数枚選択可能
- 合法手だけ選ばせるのが望ましい
- 最初は選択自体は自由でもよいが、出す時に必ずバリデーションする
- 7渡し、10捨て、12ボンバー、カード交換は追加選択 UI が必要

## CPU行動

- CPU はサーバ権威で自動行動する
- クライアントは結果だけ受け取る
- 500ms〜1200ms 程度のランダム待機を入れる
- 7渡し、10捨て、12ボンバー、カード交換時の選択もサーバ側で行う

## 通信仕様

- ルーム作成、参加、開始、手を出す、パスを通信イベントとして扱う
- ゲーム状態はサーバを正にする
- クライアントは受信状態を描画する
- 自分の番のみ入力送信可能
- 追加効果の確定結果も全員へ配信する

## 切断時の扱い

初期版推奨:
- 人間プレイヤーが切断したら一定時間は再接続待ち
- 再接続できない場合、そのプレイヤーを CPU 代打に切り替える選択肢を持つ
- まずは最低限、切断表示だけでもよい

## エラー処理

- 非合法な手は出せない
- サーバ側でも必ず再検証する
- ゲーム状態が壊れたら安全に中断またはルーム解散
- 画面側ではトーストまたはメッセージで表示

## 保存

- 初期版は永続保存不要
- ルーム単位の一時メモリでよい
- ページ再読み込み時は再接続を試みる
