# Open Questions

以下は未確定か、実装時に決め切るべき事項。

1. ルームコード方式だけにするか、URL 招待方式も入れるか
2. 切断プレイヤーを CPU 代打にするか、失格にするか
3. 縛りの厳密条件をどこまで採用するか
4. 複数ルールが同時発動した時の処理順を最終的にどう固定するか
5. 7渡し・10捨て・12ボンバー・カード交換の UI をどこまで丁寧に作るか
6. 連戦ボタンを結果画面に置くか
7. 効果音や BGM を入れるか

## 現時点の推奨

- まずはルームコード方式
- ルール変更はホストのみ
- 切断時は一定時間再接続待ち
- 複数効果はサーバ側の固定順で処理
- 追加選択 UI は modal で十分
- BGM は後回し
