# Codex Prompt (Phase 1)

以下だけを先に実装してください。  
一気に全部やらず、まずオンライン対戦の芯を安定させることを優先してください。

## Phase 1 の範囲

- ルーム作成 / 参加
- ルーム待機
- 4人固定ゲームの基本進行
- 人間2人 + CPU2人
- CPU難易度 3 段階の土台
- サーバ権威の状態管理
- 基本役
  - 単体
  - ペア
  - 3枚組
  - 4枚組
- 順位決定
- 最小 UI

## Phase 1 では入れてよいルール

- 階段
- 革命
- 8切り

## Phase 1 ではまだ後回しでよいもの

- スペ3返し
- 縛り
- 5スキップ
- 7渡し
- 10捨て
- 11バック
- 12ボンバー
- カード交換
- 豪華な UI 演出

## 必須条件

- クライアントを正にしない
- 非合法手はサーバで必ず弾く
- ゲームが 1 位〜4 位まで破綻せず進行する
- CPU が合法手しか出さない
- 処理別のフォルダ分類を守る
- 必要な reusable な補助資産が出たら `skills/` に整理する

## 作業手順の指示

1. 先にフォルダ構成を作る
2. 型を決める
3. backend のルームと状態遷移を作る
4. 基本ルールだけで 1 ゲーム完走させる
5. CPU をつなぐ
6. 最小 frontend をつなぐ
7. 起動手順を書く

## 出力してほしいもの

- frontend / backend のコード
- 起動手順
- フォルダ構成
- 主要型
- 状態遷移の説明
- テスト
