# Requirements

## プロジェクト概要

ブラウザで遊べるオンライン対戦対応の大富豪ゲームを実装する。  
基本は「友達と2人で参加」し、不足人数は CPU で補完する。  
1ゲーム内の総プレイヤー数は 4 人固定とする。

例:
- 人間2人 + CPU2人
- 人間1人 + CPU3人
- 人間3人 + CPU1人

初期リリースからオンライン対戦前提で設計する。  
そのため、ローカルだけで動けばよい構成ではなく、`ルーム作成` `参加` `同期` `切断時の扱い` まで含めて設計する。

## 目的

- 友達とオンラインで遊べる大富豪ゲームを作る
- 不足人数を CPU で補う
- CPU の強さを設定画面で 3 段階から選べるようにする
- ローカルルールを設定画面で ON/OFF できるようにする

## 対応範囲（初期版）

- タイトル画面
- ルーム作成 / 参加画面
- 対戦設定画面
- ルーム待機画面
- 対戦画面
- 追加選択モーダル
- 結果画面
- オンライン対戦の同期
- CPU 参加
- ホストによるルール設定
- ルーム内プレイヤー待機
- 連戦時のカード交換

## ゲーム人数

- 1ゲームの総人数は 4 人固定
- 人間参加者は 1〜4 人を許容する
- 不足人数は CPU で補う
- 基本ユースケースは「人間2人 + CPU2人」

## CPU難易度

設定画面で 3 段階から選択可能にする。

- Easy
- Normal
- Hard

難易度差は次で表現する。

- 強い札をどれだけ温存するか
- Joker や 2 の切り方
- 革命を狙うか
- 上がり筋を優先するか
- パス判断
- 追加効果カードの使い方

## ルール設定

対戦設定画面で、少なくとも以下を ON/OFF できるようにする。

- 階段
- 革命
- 8切り
- スペ3返し
- 縛り
- 5スキップ
- 7渡し
- 10捨て
- 11バック
- 12ボンバー
- カード交換

ホストが設定し、ルーム参加者全員へ共有する。  
ゲーム開始後は固定とする。

## UI方針

- スマホ / PC 両対応
- 初期版は PC を優先しつつ、スマホでも操作不能にならないこと
- 視認性重視
- 誤操作しにくいこと
- 通信状態が分かること

## 優先順位

1. オンライン同期が破綻しない
2. ルールが正しく動く
3. 1ゲーム最後まで遊べる
4. CPU が破綻しない
5. UI が見やすい

## 非目標

- 豪華なアニメーション
- チャット
- 観戦
- ランキング
- フレンド管理
- 課金要素

## 品質要件

- サーバを正にする
- クライアントだけで勝手に状態を決めない
- 非合法手は通さない
- ルーム参加者全員で同じ状態を見られる
- 切断時の最低限のハンドリングがある
- ロジックはテストしやすい構造にする
