# UI Specification

## UI方針

- 分かりやすさ最優先
- 自分の手札が見やすい
- 今何をすればいいか迷わない
- 誤操作しにくい
- オンライン対戦中の状態が分かる

## レイアウト

### PC

- 上部: ルーム情報 / 接続状態 / ターン情報 / ルール表示
- 中央: 場のカード
- 左右または上部: 相手3人の情報（名前、残り枚数、順位、接続状態）
- 下部: 自分の手札
- 最下部: 操作ボタン（出す、パス）

### スマホ

- 上部: 場 / ターン情報 / 接続状態
- 中部: 相手情報をコンパクト表示
- 下部: 自分の手札を横スクロールまたは折返し表示
- 固定操作バー: 出す / パス

## 必須コンポーネント

- RoomInfo
- RuleSummary
- PlayerStatus
- TableArea
- HandArea
- CardView
- SelectedActionBar
- GameLog
- ResultPanel
- ConnectionStatus
- PendingEffectModal

## 見た目ルール

- 選択中カードは少し浮かせる
- 出せない時は「出す」ボタンを無効化
- 自分の番ではない時は操作無効
- 革命中は目立つ表示を出す
- 11バック中も分かる表示を出す
- 接続切れプレイヤーは見て分かるようにする
- 8切りや 12ボンバー時は短い演出を入れてよい

## 色・演出

- 派手すぎない
- 情報の区別が付くことを優先
- CPU の思考中表示は簡易でよい
- 不要なアニメーションは入れない
