# Codex Prompt (Full)

以下の要件に従って、ブラウザで遊べるオンライン対戦対応の大富豪ゲーム初期版を実装してください。

## ゴール

- 4人固定の大富豪ゲームを実装する
- 基本は人間2人参加、不足人数は CPU で補う
- CPU難易度は設定画面で 3 段階から選べる
- 設定画面でローカルルールを ON/OFF できる
- サーバ権威でゲームを進行する
- UI とゲームロジックを分離する
- ファイルは処理別のフォルダに整理する
- 必要な reusable な補助資産があれば `skills/` フォルダを作成してその中に整理する

## 技術要件

- Frontend は React + TypeScript
- Backend は Node.js + TypeScript
- リアルタイム通信は Socket.IO または WebSocket
- Tailwind CSS を使用
- ゲームロジックは UI から分離する
- テストしやすい構造にする

## ルール要件

- 54枚（Joker 2枚）
- 4人対戦固定
- ダイヤの3スタート
- 単体、ペア、3枚組、4枚組を実装
- 階段は設定で ON/OFF
- Joker あり
- 革命は設定で ON/OFF
- 8切りは設定で ON/OFF
- スペ3返しは設定で ON/OFF
- 縛りは設定で ON/OFF
- 5スキップは設定で ON/OFF
- 5スキップは 5 の枚数分だけ次プレイヤーを飛ばす
- 7渡しは設定で ON/OFF
- 7渡しの渡し先は次のプレイヤー固定
- 10捨ては設定で ON/OFF
- 11バックは設定で ON/OFF
- 12ボンバーは設定で ON/OFF
- 12ボンバーは、Q を含む有効な手を出した後に発動プレイヤーが数字を 1 つ指定し、スート無関係で全プレイヤーがその数字の手札を強制的にすべて捨てる
- 12ボンバーでは Joker も指定可能
- 12ボンバー後も場は流さず、追加効果の解決後は通常どおり次のプレイヤーへ進む
- Joker の階段穴埋めは 1 枚まで
- 禁止上がりは採用しない
- 順位は 1 位〜4 位まで確定
- カード交換は設定で ON/OFF

## 実装上の注意

- 非合法手は出せないようにする
- クライアントから送られた手はサーバで必ず再検証する
- CPU はサーバ側で合法手一覧から選択する
- 7渡し、10捨て、12ボンバー、カード交換の追加選択処理もサーバで確定する
- 追加効果のあるルールは `pendingEffect` のような段階処理で実装する
- UI を先に豪華にしすぎない
- root に雑多なファイルを置かない
- 処理別のフォルダ分類を守る
